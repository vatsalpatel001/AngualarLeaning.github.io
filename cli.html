some of the basic Angular CLI commands
CommandDescription
ng new <project-name>Creates a new Angular project in a new folder.
ng serveStarts a development server and watches for changes in the source code.
ng buildCompiles the application into an output directory for production.
ng generate component <component-name>Generates a new component along with its files.
ng generate service <service-name>Generates a new service along with its files.
ng generate module <module-name>Generates a new module along with its files.
ng generate directive <directive-name>Generates a new directive along with its files.
ng generate pipe <pipe-name>Generates a new pipe along with its files.
ng generate class <class-name>Generates a new class along with its files.
ng testRuns unit tests using Karma and Jasmine.
ng lintRuns the linter to check the code for linting errors.
ng build --prodBuilds the application for production, with optimization and minification.
ng helpDisplays help information about Angular CLI commands.
Here are some additional Angular CLI commands
CommandDescription
ng generate enum <enum-name>Generates a new TypeScript enumeration.
ng generate interface <interface-name>Generates a new TypeScript interface.
ng generate guard <guard-name>Generates a new route guard.
ng generate component <component-name> --inline-templateGenerates a new component with an inline template.
ng generate component <component-name> --inline-styleGenerates a new component with inline styles.
ng generate component <component-name> --skip-testsGenerates a new component without test files.
ng add <package-name>Adds a package or a library to the current project.
ng updateUpdates the application and its dependencies.
ng config <config-option>Retrieves or sets Angular configuration values.
ng serve --openStarts the server and opens the application in a browser.
ng build --watchCompiles the application and watches for changes.
ng test --watchRuns unit tests and watches for changes to re-run the tests.
ng e2eRuns end-to-end tests using Protractor.
ng lint <project-name>Runs the linter on a specific project.
ng doc <keyword>Opens the official Angular documentation for a specific keyword in the browser.
